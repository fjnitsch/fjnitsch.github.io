  • We have to make prior assumptions explicit
  • We can use hierarchical models

Cumulative Prospect Theory

  • Enhancement of Subjective Expected Utility Theory (SEU)
  • Can account for a series of deviations from SEU
    • Reflection Effect
    • Inconsistent risk aversion/risk seeking
  • Both values and probabilities are transformed domain-dependently
  • The general form of the \(SV\) (similar to SEU) of a prospect \(O\) is given by:

\[SV(O) = \pi(p_i)u(x_i)\] \[O: x_i \leftarrow p_i\]

Component 1: Domain-dependent Utility Function

The utility function is domain dependent:

\[ u(x)= \begin{cases} x^\alpha \leftarrow x\geq0 \\ -\lambda(-x)^\beta \leftarrow x < 0 \end{cases} \] with

\[ 0\leq\alpha,\beta\leq1 \\ \lambda > 0 \]

  • \(\alpha\) and \(\beta\) shape the curvature of the function
  • \(\lambda\) specifies loss aversion (larger values mean more weight assigned to losses)
  • We constrain \(\lambda\) to be smaller than 10 for computational simplicity

Component 2: Probability Weights

Probability weights are also domain dependent

\[ \pi(p_i) = \frac{p_i^{c(x_i)}}{(p_i^{c(x_i)}+[1-p_i]^{c(x_i)})^{1/c(x_i)}} \] with

\[ c(x)= \begin{cases} \gamma \leftarrow x\geq0 \\ \delta \leftarrow x < 0 \end{cases} \] and

\[ 0\leq\gamma,\delta\leq1 \]

  • as long as \(c<1\), small probabilities are overestimated and large probablities are underestimated

Visualisation of Utility Function and Probability Weights

Component 3: Luce Choice Rule

  • Original CPT is deterministic, highest SEU option is always chosen
  • Problem: Behaviour is noisy
  • We need an error model to account for behaviour
  • Simplest solution: Some form of simple scalability
    • Choice probability is a monotone function of SEU difference
  • Example: logistic Luce choice rule

\[ Pr(O_A,O_B) = \frac{1} {1 + e^{\phi [SV(O_B)-SV(O_A)]}} \] with

\[ \phi \geq 0 \]

  • \(\phi\) is a sensitivity parameter
  • For \(\phi=0\), choice behaviour is uniform random (\(Pr(O_A,O_B)=0.5\))
  • As \(\phi\) grows larger, choice becomes more sensitve for SEU differences

Visualisation of logistic Luce Choice Rule

Component 4: Bernoulli Trials

Finally we model the decision between two prospects as a Bernoulli trial

\(R_{O_A, O_B} \stackrel{}{\sim} dbern(Pr(O_A, O_B))\)

  • dbern is the Bernoulli probability density function

Priors as an expression about our beliefs of the probability distribution of the model parameters

  • We model subjects with individual parameters
  • But: Subjects are similar
    • Parameters are drawn from joint distribution
  • We assume that parameters are stable
    • Implies no learning
    • Implies stable preferences

Transformation of parameters for estimation

Given are the following constraints for all probabilistic CPT parameters:

\[ \alpha, \beta, \gamma, \delta \in[0,1]\\ \lambda, \phi \in[0, \infty] \]

To faciliate hierarchical modeling, we probit transform \(\alpha, \beta\), \(\gamma\) and \(\delta\)

Transformed parameters cover the entire real line

\[ \alpha^\phi \in [\infty_-,\infty_+] = \Theta^{-1}(\alpha)\\ \beta^\phi \in [\infty_-,\infty_+] = \Theta^{-1}(\beta)\\ \gamma^\phi \in [\infty_-,\infty_+] = \Theta^{-1}(\gamma)\\ \delta^\phi \in [\infty_-,\infty_+] = \Theta^{-1}(\delta) \] Further, we log transform \(\lambda\) and \(\phi\)

\[ \lambda^{log} \in [0,\infty_+] = log(\lambda) \\ \phi^{log} \in [0,\infty_+] = log(\phi) \]

Specification of Subject Level Priors

We assume the transformed parameters to be drawn from independent group-level normal distributions:

\[ \alpha^\phi_i \stackrel{}{\sim} dnorm(\mu_\alpha, \sigma_\alpha)\\ \beta^\phi_i \stackrel{}{\sim} dnorm(\mu_\beta, \sigma_\beta)\\ \gamma^\phi_i \stackrel{}{\sim} dnorm(\mu_\gamma, \sigma_\gamma)\\ \delta^\phi_i \stackrel{}{\sim} dnorm(\mu_\delta, \sigma_\delta)\\ \] We assume the rest of the parameters to be drawn from independent group-level log-normal distributions:

\[ \lambda^{log}_i \stackrel{}{\sim} dlnorm(\mu_\lambda, \sigma_\lambda)\\ \phi^{log}_i \stackrel{}{\sim} dlnorm(\mu_\phi, \sigma_\phi) \]

Specification of Group Level Priors

Mostly uninformative priors \[ \mu_\alpha, \mu_\beta, \mu_\gamma, \mu_\delta \stackrel{}{\sim} dnorm(0,1)\\ \sigma_\alpha, \sigma_\beta, \sigma_\gamma, \sigma_\delta \stackrel{}{\sim} dunif(0, 10)\\ \]

Priors for \(\lambda\) and \(\phi\) restrict eligible parameter values to a range of \([0.1, 5]\), based on authorsexperience

\[ \mu_\lambda, \mu_\phi \stackrel{}{\sim} dunif(-2.30, 1.61)\\ \sigma_\lambda, \sigma_\phi \stackrel{}{\sim} dunif(0, 1.13) \]

Model Specification in JAGS

# Fits Cumulative Prospect Theory
# Data from Rieskamp (30 paritipants, 60 gamble-pairs 
# with positive outcomes, 60 gamble-pairs with negative
# outcomes, and 60 gamble-pairs with mixed outcomes)


model
{


  # The model assumes that every participant has a unique parameter-value 
  # on each of the six parameters of hierarchical-CPT 
  # (see, Nilsson, Rieskamp, & Wagnemakers, 2010).
  # Furthermore, it assumes that the individual values on parameter-dimension X 
  # are sampled from the hyperdistribution for X. This has to be defined in the model code.
  
    
  for (j in 1:30) {
  
    # We start by defining that participant j has a unique parameter-value 
    # on each of the six parameters: alpha, beta, gamma, delta, lambda, and luce 
    # (the choice-rule parameter).
  
    alpha[j] <- phi(alpha_phi[j])
    beta[j] <- phi(beta_phi[j])
    gamma[j] <- phi(gamma_phi[j])
    delta[j] <- phi(delta_phi[j])   
    luce[j]   <- exp(lluce[j]) 
    lambda[j] <- exp(llambda[j])
        
    # We put group-level normal distributions on the individual parameters:
  
    alpha_phi[j] ~ dnorm(mu_phi_alpha,tau_phi_alpha) T(-3, 3)
    beta_phi[j] ~ dnorm(mu_phi_beta,tau_phi_beta) T(-3, 3)
    gamma_phi[j] ~ dnorm(mu_phi_gamma,tau_phi_gamma) T(-3, 3)
    delta_phi[j] ~ dnorm(mu_phi_delta,tau_phi_delta) T(-3, 3)
    lluce[j] ~ dnorm(lmu_luce, ltau_luce)
    llambda[j] ~ dnorm(lmu_lambda, ltau_lambda)
  }
  
  # Here priors for the hyperdistributions are defined:
  mu_phi_alpha ~ dnorm(0,1)
  tau_phi_alpha <- pow(sigma_phi_alpha,-2)
  sigma_phi_alpha ~ dunif(0,10)
  
  mu_phi_beta ~ dnorm(0,1)
  tau_phi_beta <- pow(sigma_phi_beta,-2)
  sigma_phi_beta ~ dunif(0,10)
  
  mu_phi_gamma ~ dnorm(0,1)
  tau_phi_gamma <- pow(sigma_phi_gamma,-2)
  sigma_phi_gamma ~ dunif(0,10)
  
  mu_phi_delta ~ dnorm(0,1)
  tau_phi_delta <- pow(sigma_phi_delta,-2)
  sigma_phi_delta ~ dunif(0,10)
    
  lmu_lambda    ~ dunif(-2.3, 1.61) 
  ltau_lambda  <- pow(lsigma_lambda,-2)
  lsigma_lambda ~ dunif(0,1.13)    
  
  lmu_luce    ~ dunif(-2.3, 1.61)   
  ltau_luce  <- pow(lsigma_luce,-2)
  lsigma_luce ~ dunif(0,1.13)       
  
    
  # To obtain the mean of the hyper distribution on the wanted scale:
  mu_alpha <- phi(mu_phi_alpha) 
  mu_beta <- phi(mu_phi_beta)
  mu_gamma <- phi(mu_phi_gamma) 
  mu_delta <- phi(mu_phi_delta) 
  mu_lambda <- exp(lmu_lambda)
  mu_luce   <- exp(lmu_luce)

  # It is now time to define how the model should be fit to data.
  # Subject-loop
  for (j in 1:30) {
  
    # Item-Loop, positive gambles,gamble A
    for (i in 1:60) {
    
      # j:s subjective value of outcome 1 in gamble A of gamble-pair i, v_x_a[i,j], equals:
      v_x_a[i,j] <- pow(prospects_a[i,1],alpha[j])
      
      # where prospects_a[i,1] = outcome 1 and alpha[j] = j:s value on the alpha parameter
      # j:s subjective value of outcome 2 in gamble A
      v_y_a[i,j] <- pow(prospects_a[i,3],alpha[j])       
      
      # j:s subjective probability of outcomes 1, w_x_a[i,j], and 2, w_y_a[i,j].
      w_x_a[i,j] <- pow(prospects_a[i,2],gamma[j]) / pow(z_a[i,j],(1/gamma[j])) 
      w_y_a[i,j] <- pow(prospects_a[i,4],gamma[j]) / pow(z_a[i,j],(1/gamma[j])) 
      z_a[i,j]   <- pow(prospects_a[i,2],gamma[j]) + pow(prospects_a[i,4],gamma[j]) 
  
      # j:s subjective value of gamble A in gamble-pair i
      Vf_a[i,j]  <- w_x_a[i,j] * v_x_a[i,j] + w_y_a[i,j] * v_y_a[i,j]
    }
    
    # Item-Loop, positive gambles,gamble B
    for (i in 1:60) {
      v_x_b[i,j] <- pow(prospects_b[i,1],alpha[j])       
      v_y_b[i,j] <- pow(prospects_b[i,3],alpha[j])
      
      w_x_b[i,j] <- pow(prospects_b[i,2],gamma[j]) / pow(z_b[i,j],(1/gamma[j])) 
      w_y_b[i,j] <- pow(prospects_b[i,4],gamma[j]) / pow(z_b[i,j],(1/gamma[j])) 
      z_b[i,j]   <- pow(prospects_b[i,2],gamma[j]) + pow(prospects_b[i,4],gamma[j]) 
  
      Vf_b[i,j]  <- w_x_b[i,j] * v_x_b[i,j] + w_y_b[i,j] * v_y_b[i,j]
    }
   
    # Item-Loop, positive gambles,choice-rule
    for (i in 1:60) {
      # It is defined that j:s choice for gamble-pair i is sampled from a Bernoulli-distribution 
      # with p = binval[i,j].
      # The exponential choice-rule is used to compute binval[i,j] 
      # (in this case: binval[i,j] = probability of B-choice).
        
        binval[i,j] <- (1)/(1+exp((-1*luce[j])*(Vf_b[i,j]-Vf_a[i,j])))
        rawdata[i,j] ~ dbern(binval[i,j])
     }
     
    # Item-Loop, negative gambles,gamble A
    for (i in 61:120) {
      v_x_a[i,j] <- (-1) * pow((-1 * prospects_a[i,1]),beta[j])       
      v_y_a[i,j] <- (-1) * pow((-1 * prospects_a[i,3]),beta[j])       
  
      w_x_a[i,j] <- pow(prospects_a[i,2],delta[j]) / pow(z_a[i,j],(1/delta[j])) 
      w_y_a[i,j] <- pow(prospects_a[i,4],delta[j]) / pow(z_a[i,j],(1/delta[j])) 
      z_a[i,j]   <- pow(prospects_a[i,2],delta[j]) + pow(prospects_a[i,4],delta[j]) 
  
      Vf_a[i,j]  <- w_x_a[i,j] * v_x_a[i,j] + w_y_a[i,j] * v_y_a[i,j]
    }
     
    # Item-Loop, negative gambles,gamble B
    for (i in 61:120) {
      v_x_b[i,j] <- (-1) * pow((-1 * prospects_b[i,1]),beta[j])       
      v_y_b[i,j] <- (-1) * pow((-1 * prospects_b[i,3]),beta[j])       
  
      w_x_b[i,j] <- pow(prospects_b[i,2],delta[j]) / pow(z_b[i,j],(1/delta[j])) 
      w_y_b[i,j] <- pow(prospects_b[i,4],delta[j]) / pow(z_b[i,j],(1/delta[j])) 
      z_b[i,j]   <- pow(prospects_b[i,2],delta[j]) + pow(prospects_b[i,4],delta[j]) 
  
      Vf_b[i,j]  <- w_x_b[i,j] * v_x_b[i,j] + w_y_b[i,j] * v_y_b[i,j]
    }
    
    # Item-Loop, negative gambles,choice-rule
    for (i in 61:120) {
      binval[i,j] <- (1)/(1+exp((-1*luce[j])*(Vf_b[i,j]-Vf_a[i,j])))
        rawdata[i,j] ~ dbern(binval[i,j])
    }
    
    # Item-Loop, mixed gambles,gamble A
    for (i in 121:180) {
      v_x_a[i,j] <- pow(prospects_a[i,1],alpha[j])       
      v_y_a[i,j] <- (-1 * lambda[j]) * pow((-1 * prospects_a[i,3]),beta[j])       
  
      w_x_a[i,j] <- pow(prospects_a[i,2],gamma[j]) / pow(z_a[i,j],(1/gamma[j])) 
      w_y_a[i,j] <- pow(prospects_a[i,4],delta[j]) / pow(z_a[i,j],(1/delta[j])) 
      z_a[i,j]   <- pow(prospects_a[i,2],gamma[j]) + pow(prospects_a[i,4],delta[j]) 
  
      Vf_a[i,j]  <- w_x_a[i,j] * v_x_a[i,j] + w_y_a[i,j] * v_y_a[i,j]
    }
    
    # Item-Loop, mixed gambles,gamble B
    for (i in 121:180) {
      v_x_b[i,j] <- pow(prospects_b[i,1],alpha[j])       
      v_y_b[i,j] <- (-1 * lambda[j]) * pow((-1 * prospects_b[i,3]),beta[j])
      
      w_x_b[i,j] <- pow(prospects_b[i,2],gamma[j]) / pow(z_b[i,j],(1/gamma[j])) 
      w_y_b[i,j] <- pow(prospects_b[i,4],delta[j]) / pow(z_b[i,j],(1/delta[j])) 
      z_b[i,j]   <- pow(prospects_b[i,2],gamma[j]) + pow(prospects_b[i,4],delta[j]) 
  
      Vf_b[i,j]  <- w_x_b[i,j] * v_x_b[i,j] + w_y_b[i,j] * v_y_b[i,j]
    }
    
    # Item-Loop, mixed gambles,choice-rule
    for (i in 121:180) {
      binval[i,j] <- (1)/(1+exp((-1*luce[j])*(Vf_b[i,j]-Vf_a[i,j])))
      rawdata[i,j] ~ dbern(binval[i,j])
    }
  }   
}

Fit Model to Data

# Load information about the 180 gamble-pairs used in Rieskamp (2008). GambleA.txt and GambleB.txt are 
# structured as follows: value of outcome 1 (column 1), probability of outcome 1 (column 2), value of outcome 2 # (column 3), probability of outcome 2 (column 4) (gambles in rows).
prospects_b <- as.matrix(read.table("NilssonEtAl2011_code/GambleB.txt"))
prospects_a <- as.matrix(read.table("NilssonEtAl2011_code/GambleA.txt"))

# Load data (choice made by the first participant when presented the second gamble-pair is saved in
# column 1 row 2)
rawdata <- as.matrix(read.table("NilssonEtAl2011_code/Rieskamp_data.txt"))

# Define what information that should be passed on to WinBugs
data  = list("prospects_a" = prospects_a, 
             "prospects_b" = prospects_b,
             "rawdata" = rawdata)

# Define initial values for variables (variables are defined in: cpt_hierarchical_model.txt)
inits = function()
{
  
  list(mu_phi_alpha = 0.7, sigma_phi_alpha = 1, mu_phi_beta = 0.7,
       sigma_phi_beta = 1,
       mu_phi_gamma = 0.7, sigma_phi_gamma = 1, mu_phi_delta = 0.7,
       sigma_phi_delta = 1,
       lmu_lambda = 0, lsigma_lambda = 0.5, lmu_luce = 0)
  
}

# Define the variables of interest. 
monitored_parameters = c("alpha", "mu_phi_alpha", "mu_alpha", "sigma_phi_alpha", 
               "beta", "mu_phi_beta", "mu_beta", "sigma_phi_beta", 
               "gamma", "mu_phi_gamma", "mu_gamma", "sigma_phi_gamma", 
               "delta", "mu_phi_delta", "mu_delta", "sigma_phi_delta", 
               "lambda", "lmu_lambda", "mu_lambda", "lsigma_lambda", 
               "luce", "lmu_luce", "mu_luce", "lsigma_luce")

# Run MCMC
mcmcfin <- run.jags(model = "CPT_model.j",
                    data = data, 
                    inits = inits,
                    n.chains = 2,
                    sample = 10000,
                    adapt = 1000,
                    burnin = 1000,
                    thin = 10,
                    monitor = monitored_parameters,
                    method = "parallel"
                    )

# Store posterior means for later
posterior_means <- as.matrix(apply(as.matrix(as.mcmc.list(mcmcfin)), 2, mean))
write.csv(posterior_means, "posterior_means.csv")
